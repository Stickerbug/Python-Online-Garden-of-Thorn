# GTN 卡牌描述规范

本文件规定 GTN 中文卡牌、标签、状态、配装及相关术语的描述写法。目标是在不改变现有语言风格的前提下，让文本尽可能短、准确且无歧义。

卡牌的实际逻辑、伤害预测、战斗日志和术语说明必须表达同一规则。不能只修改卡面而保留不一致的实现。

## 1. 卡牌资料格式

提交卡牌时使用以下顺序：

```text
中文名称 English Name
xE xM T/B/R/G
x张
标签或特殊效果
效果描述
趣味描述
```

- `T/B/R/G` 分别代表 Thorn、Bloom、Root、Guard。
- 数量表示抽牌权重；`0张`表示正常抽牌无法获得。
- 标签与特殊效果单独填写，不混入效果描述。
- 中文效果以 `effect_text` 为唯一数据源，不另建 `effect_text_cn` 等重复字段。
- 图片、名称、消耗、类型和权重必须与实际数据一致。

## 2. 基本写法

### 2.1 句式

- 优先使用“对象 + 动作 + 结果”或“动作 + 对象 + 结果”。
- 有目标的主动效果优先写“对目标……”。
- 只作用于自己时写“自己……”。
- 同一句中的连续结算用逗号；相对独立的效果用分号。
- 多个规则按实际结算顺序书写。
- 不写“可能”“大概”“类似”等不确定表述。
- 不使用“对应规则”“正常生效”等无法独立解释规则的省略语。

示例：

```text
对目标造成8[[icon:D]]；命中时，对目标施加2层流血
回复目标5[[icon:M]]
自己回合开始时，对目标施加1层虚弱
```

### 2.2 数字、符号与标点

- 层数、次数、消耗和数值统一使用阿拉伯数字。
- 乘法使用 `×`，范围使用 `0~3`，比较使用 `≤`、`≥`。
- 百分数后需要说明取整方式时，写“向上取整”或“向下取整”。
- 中文效果使用全角中文标点。
- 括号只用于公式、补充限制或取整方式，不把主要规则藏进括号。
- H、E、M、D、P、F 使用项目支持的图标标记，例如 `[[icon:H]]`、`[[icon:D]]`。
- 多段伤害写为 `2[[icon:D]]×3`，不要写为 `2×3[[icon:D]]`。

### 2.3 标签不重复

卡牌数据中已经带有的标签或特殊效果，不在效果描述中重复说明。只有以下情况可以在效果描述中提到：

- 获得、移除或改变标签；
- 检查某标签作为条件；
- 说明该牌对标签通常规则的例外。

例如卡牌本身带有“放逐”，效果描述中不再写“放逐”；但“若此牌无放逐，则……”可以保留。

## 3. 对象定义

| 用词 | 含义 |
| --- | --- |
| 自己 | 当前效果的使用者或来源玩家 |
| 目标 | 当前效果实际选中的玩家或卡牌 |
| 装备拥有者 | 使用该装备、且装备显示在其装备栏的玩家 |
| 装备目标 | 装备效果实际作用的玩家，可与装备拥有者不同 |
| 攻击者 | 造成当前伤害的玩家 |
| 触发者 | 使装备或响应条件成立的玩家 |
| 所有玩家 | 当前对局中的全部未阵亡玩家；若包括阵亡玩家，必须明确写出 |
| 所有牌 | 手牌、抽牌堆、弃牌堆和放逐区中的牌，不包括装备 |
| 所有牌与装备 | 上述所有牌以及装备栏中的装备 |

- 能选择阵亡玩家时必须明确说明。
- “对方”“敌方玩家”“队友”不可混用；多人模式下按实际范围选择。
- 装备给目标时，装备仍属于装备拥有者；界面使用“装备名→玩家名”表达效果目标。
- 装备效果中的“自己”若容易与装备目标混淆，应改写为“装备拥有者”或“目标”。

## 4. 时点与条件

必须明确效果发生的时点：

- `使用时`：牌被打出并进入结算时；
- `装备时`：牌成功进入装备栏时；
- `命中时`：该次伤害实际命中目标后；
- `造成伤害时`：该次伤害造成了实际伤害后；
- `受到伤害时`：目标实际承受该类伤害后；
- `回合开始时`、`回合结束时`：对应玩家的回合时点；
- `抽取时`：牌进入手牌时；
- `进入弃牌堆时`、`被放逐时`、`被摧毁时`：对应区域移动完成时。

条件写在结果之前：

```text
若自己[[icon:H]]≤30%，则本次伤害翻倍
已装备1回合时，可触发：摧毁本装备；选择一个目标……
```

不要用“之后”“下回合”等模糊说法代替明确对象。应写“目标下回合开始时”“自己下个回合结束时”。

## 5. 伤害、回复与资源

- `[[icon:D]]` 表示物理伤害。
- 电伤是一种魔法伤害，使用电伤图标，不写成物理伤害。
- `[[icon:P]]` 与 `[[icon:F]]` 分别表示中毒和灼烧层数，不等同于即时伤害。
- “命中”要求伤害通过相应免除规则并命中；“造成伤害”还要求实际伤害大于0。
- 依赖实际伤害的回复、反伤或施加效果，必须按最终实际伤害计算。
- 回复写明对象与资源，例如“回复目标8[[icon:H]]”“回复自己3[[icon:M]]”。
- 改变上限时写“[[icon:E]]上限+5”，仅回复时写“回复5[[icon:E]]”。
- 多段伤害、反伤和附加效果必须明确是每段触发还是整张牌只触发一次。

伤害预测应与实际结算使用相同规则，包括目标、护甲、状态免疫、闪避、反制、多子瓣、自伤、回复和伤害上限。

## 6. 卡牌区域、选择与生成

- 移动卡牌必须写清来源与去向，例如“从目标弃牌堆选择1张牌，将其置入目标抽牌堆顶”。
- “弃置”表示主动从手牌进入弃牌堆；打出后自然进入弃牌堆不算主动弃置。
- “抽取”只指从抽牌堆抽牌；直接加入手牌不算抽取。
- “摧毁”只用于装备；普通牌使用“弃置”“放逐”或“置入某牌堆”。
- 选择数量写“选择至多3张”“选择1张”，并说明没有合法选项时是否取消。
- 随机效果必须写清随机范围。
- 生成或展示具体卡牌时使用 Chip：

```text
[[card:魔法球|flag=sprout|flag=symbiosis]]
```

- Chip 中的标签属于生成牌实例，不再在 Chip 外重复。
- 生成的牌是否保留原牌标签、特殊效果、放逐或唯一，必须按实际规则明确。
- 自动打牌必须说明目标、费用、选择流程、无法使用时的处理以及是否触发原牌效果。

## 7. 标签与特殊效果

### 7.1 定义

- 标签是卡牌携带的规则标记，例如精准、放逐、共生、唯一。
- 特殊效果是卡牌实例上可带层数或数值的修饰，例如聚变、裂变、迅捷、魔力迅捷、威力、暂时沉重。
- 卡牌描述可以用“特殊效果”统称这些实例修饰；无层数的普通标签不称为“特殊效果层数”。

### 7.2 常用规则

| 名称 | 规范含义 |
| --- | --- |
| 迅捷:X | E花费-X |
| 魔力迅捷:X | M花费-X |
| 暂时迅捷:X | E花费-X，打出后清除 |
| 暂时沉重:X | E花费+X，打出后清除 |
| 暂时魔力沉重:X | M花费+X，打出后清除 |
| 威力:X | 造成的物理伤害增加；多段物理伤害按术语规则分配，打出后清除 |
| 不选择目标 | 使用时不进入目标选择 |
| 队伍限定 | 只在每队至少有2名玩家的模式中出现；单人训练场可以出现但不生效 |
| 崇高 | 无法被除使用外的行为选中 |

- 术语已有完整说明时，卡面不再复制说明。
- 内部控制标记、兼容标记和脚本标记不得展示给玩家。
- 同一规则只展示一次，不同时显示原版名称、模组名称或内部 ID。

## 8. 状态描述

卡牌只需写“对目标施加X层状态”。状态的完整规则放在状态术语中，不在每张牌上重复。

新增状态必须说明：

1. 层数如何影响效果；
2. 生效对象与生效时点；
3. 如何减少、消耗或清除；
4. 状态免疫存在时如何处理；
5. 是否影响伤害预测或信息显示。

以下容易混淆的现行规则必须按此表达：

- **状态免疫**：状态仍可获得层数，并按各自规则减少或消耗，但不产生效果。护甲不是状态。
- **流血**：自己打出攻击牌时，受到等同于流血层数的魔法伤害；自己回合结束时，层数向下取整减半。
- **回合回复:X;Y**：出现时立即回复Y[[icon:H]]并使X-1；自己回合开始伤害结算后，回复Y[[icon:H]]并使X-1；X=0时移除。
- **魔力回合回复:X;Y**：出现时立即回复Y[[icon:M]]并使X-1；自己回合开始伤害结算后，回复Y[[icon:M]]并使X-1；X=0时移除。
- **中毒**：自己回合开始时结算伤害，再按规则减少层数；状态免疫时不造成伤害，但仍减少层数。
- **灼烧**：自己回合开始时结算伤害，默认不主动减少；状态免疫时不造成伤害。

完整状态定义以游戏内术语数据为准。本文件不重复维护整份状态图鉴。

## 9. 装备、触发与反制

### 9.1 装备

- 装备效果写明是作用于装备拥有者、装备目标、攻击者还是触发者。
- 装备的回合开始或结束效果写明是哪名玩家的回合。
- 装备摧毁、装备护甲、自毁触发和离场清理必须使用统一的装备生命周期。
- 同名装备存在多个实例时，每个实例独立保存层数、目标和装备回合数；摧毁一个实例不得清除其他实例的效果。

触发格式：

```text
已装备1回合时，可触发：摧毁本装备；选择一个目标……
```

### 9.2 反制

- 响应条件统一放在效果描述末尾。
- 效果与响应条件之间使用两个半角空格：

```text
使所响应的技能牌失效  响应：敌方使用技能牌
```

- 条件必须写清响应谁、响应什么行为，以及是“被作为目标”还是“受到伤害”。
- “所响应的牌失效”和“所响应的牌结算后进入放逐区”是不同规则，不能混写。
- 反制窗口、可支付条件和结算顺序必须与描述一致。

## 10. 配装、子瓣与动态文本

- 配装描述使用与卡牌相同的对象、时点、图标和 Chip 规则。
- 多子瓣牌的卡面、升级图、伤害预测和实际命中次数必须来自同一份子瓣数据。
- “不受多重瓣影响”“不会损失裂变层数”等例外必须明确写出。
- 动态变化后的卡牌描述应展示当前实例效果；图鉴和未被修改的抽牌列表展示原始定义。
- 术语展示应根据牌当前实际拥有的标签、特殊效果和状态引用生成，不能仅靠名称或描述文本猜测。

## 11. 趣味描述

- 趣味描述与规则描述分开存放。
- 不承担规则解释，不写未实现机制。
- 可以口语化，但应简短并符合卡牌主题。
- 趣味描述不使用规则涂色。

## 12. 提交前检查

- [ ] 中文名、英文名、类型、消耗、权重和图片正确。
- [ ] 效果描述与实际逻辑、预测和日志一致。
- [ ] 主体、目标、时点、区域和取整方式明确。
- [ ] 多段效果明确按段还是按牌触发。
- [ ] 已有标签没有在效果描述中重复。
- [ ] 新增牌使用 Chip，图标标记能够正常渲染。
- [ ] 反制条件位于末尾，前面有两个半角空格。
- [ ] 新状态有层数、生效、衰减和状态免疫规则。
- [ ] 装备目标和装备拥有者没有混淆。
- [ ] 1v1、2v2、训练场、观战和回放中的信息一致。
- [ ] 内部 ID、内部标签和未翻译文本不会显示给玩家。
- [ ] 趣味描述不包含必要规则。

## 13. 卡牌模板

```text
示例卡牌 Example Card
xE xM T/B/R/G
x张
标签或特殊效果
对目标……；满足条件时，……  响应：……
趣味描述。
```

新增自定义内容时附加：

```text
新增标签：
示例标签：完整规则说明

新增状态：
示例状态：生效规则；减少或清除规则；状态免疫时的规则
```
